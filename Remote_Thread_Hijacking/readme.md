## Overview
In this lab, we cover Remote Thread Hijacking technique, Remote Thread Hijacking is a method of injecting code into a process by hijacking an existing thread in that process. Unlike traditional code injection methods (e.g., using CreateRemoteThread or NtCreateThreadEx), this technique manipulates an already-running thread to execute malicious payloads. This makes detection harder, as no new thread creation events are logged, and it avoids common anti-malware defenses that monitor thread creation.

## Steps
1. Identify a Target Process and Thread
2. Suspend the Target Thread
3. Get Thread Context
4. Allocate Memory in the Target Process
5. Write the Payload into Allocated Memory
6. Modify the Thread's Execution Context
7. Resume the Thread

Creating a snapshot of all threads in the system using `CreateToolhelp32Snapshot`. It then iterates through these threads with `Thread32Next` to find one belonging to the specified process (pid). Upon locating the target thread, it opens a handle to this thread using OpenThread and suspends its execution with SuspendThread. The context of the suspended thread is retrieved with `GetThreadContext`, allowing the code to modify the instruction pointer `(RIP)` to point to the malicious payload located at `rBuffer`.
